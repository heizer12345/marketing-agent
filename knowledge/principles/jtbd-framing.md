# Jobs-To-Be-Done (JTBD) framing

## The frame
> "When [situation], I want to [motivation], so I can [outcome]."

Customers don't buy products. They hire a product to do a job. Anchor copy on the job, the trigger situation, and the outcome — not the feature.

## How to use it
1. Identify the **triggering situation** (the moment the buyer realizes they need the job done).
2. Name the **motivation** in their words (what they're actively trying to do).
3. End in the **outcome** they're hiring you to deliver.

Then write copy that maps to that JTBD statement directly.

## Rules of thumb
- The trigger is usually a moment, not a state. ("When my last supplier ghosted on a $40k order...")
- Avoid feature-first copy: "AI matching" is a feature, not a job.
- One JTBD per page. Multiple jobs = different segments = different landing pages.

## Sourcy example JTBDs

- "When my last supplier ghosted mid-PO, I want a verified backup factory within the week, so I can ship on schedule."
- "When my brand portfolio added a new SKU line, I want to source it without standing up another procurement function, so I can scale without headcount."
- "When my Alibaba RFQ replies all looked sketchy, I want someone who's vetted the factory in person, so I can stop worrying about getting scammed."

## Template prompt
"Write a {asset_type} for ICP {icp} using JTBD framing. Triggering situation: {trigger}. Motivation: {motivation}. Outcome: {outcome}. Lead with the situation, end with the outcome. No feature-first copy. Sourcy voice."
